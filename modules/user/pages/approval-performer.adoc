= Задание на согласование

Согласование документа выполняется с помощью задания _На согласование_.

include::partial$excerpts.adoc[tags=incomingFolder]
+
. Откройте полученное задание с описанием _Согласование документов_ в состоянии `Не начато`.
+
.Карточка задания _На согласование_, открытая у согласующего
image::approval-performer.png[Карточка задания На согласование, открытая у согласующего]
+
include::partial$approval.adoc[tags=fields]
+
. Ознакомьтесь с составом xref:approval-files.adoc[согласуемых и дополнительных файлов].
. xref:approval-files.adoc#addVersion[Добавьте собственную версию согласуемого файла].
. xref:approval-files.adoc#addComment[Добавьте комментарий к версии].
+
TIP: Особенности формирования версий файлов приведены в пункте xref:approval-files.adoc#versioningRules[Правила формирования версий файлов при согласовании].
+
. Если требуется, создайте подчиненные задания согласования для других пользователей.
+
****
Подчиненные задания позволяют получить от других сотрудников информацию, необходимую для принятия решения по согласованию. Работа с подчиненными заданиями рассмотрена в пункте xref:approval-subtasks.adoc[Подчиненные задания согласования].
****
+
. Примите решение по согласованию. Нажмите соответствующую кнопку принятия решения на xref:cards-terms.adoc#cards-operations[панели операций].
+
****
[horizontal]
Согласовано::
Положительное решение. Документ будет согласован и передан следующему исполнителю.
+
Согласовано с замечаниями::
Условно-положительное решение. Документ будет согласован и направлен следующему участнику согласования с комментарием к решению.
+
Отказано::
Отрицательное решение. Документ не будет согласован, карточка будет отправлена следующему исполнителю, либо произойдет переход к новому циклу согласования.

[horizontal]
.Также могут быть доступны дополнительные варианты:
Делегировать::
Делегирует задание другому исполнителю.

Добавить согласующих::
Добавляет новых участников в согласование. Инструкция по работе с дополнительными согласующими приведена в xref:approval-additional-approvers.adoc[Дополнительные согласующие].

Результат согласования будет отображен в компоненте _Ход согласования_.
****
+
. Введите причину, по которой было принято соответствующее решение в появившемся окне, если принятое решение по согласованию предполагает ввод комментария.
. Нажмите кнопку *Подтвердить*, чтобы добавить комментарий.
+
.Добавление комментария к решению
image::approval-decision-comment.png[Добавление комментария к решению]
+
****
Если согласующим был добавлен комментарий к загруженной версии согласуемого файла, этот комментарий будет автоматически скопирован в поле комментария к решению.

Задание будет завершено. Описание решения, принятого согласующим, появится в верхней части экрана.

.Интерфейс карточки завершённого задания на согласование
image::approval-approved.png[Интерфейс карточки завершённого задания на согласование]
****
